## 1. 项目背景与愿景 (Project Background & Vision)

### 1.1 背景
随着数字化浪潮的推进，非物质文化遗产（非遗）面临着传承人老龄化、技艺难以量化传授、年轻群体接触门槛高等痛点。传统的非遗保护多停留在图文记录或视频存档层面，缺乏深度的交互性和技能复原能力。

本项目紧扣“中国古代物理/中华优秀传统文化”的大赛主题，利用前沿的 **多智能体协作 (Multi-Agent)** 和 **多模态交互 (Multimodal Interaction)** 技术，构建一个能够“看懂手势、听懂提问、生成创意”的数字化非遗导师。

### 1.2 产品愿景
打造一款**“能看、能听、会画”的 AI 非遗传承助手**。
通过 AI Agent 填补非遗技艺中“言传身教”的数字化空白，实现低门槛、高互动的非遗传承教学，让每一位用户都能成为“数字传承人”。

## 3. 使用场景 (Usage Scenarios)

### 场景一：沉浸式技艺学习 (Learning)
**用户**：大二学生小李，选修了“苏绣艺术欣赏”课程。
**流程**：
1.  小李打开 App，选择“苏绣-平针绣”课程。
2.  **视觉导师 Agent** 启动摄像头，识别小李的握针手势。
3.  小李尝试下针，屏幕实时显示骨骼关键点，并提示：“手指捏合力度过大，请放松，角度需倾斜 45 度”。
4.  小李调整动作，系统显示“动作标准”，并记录一次有效练习。

### 场景二：创意设计与复原 (Designing)
**用户**：文创设计师 Sarah，正在设计一款“青花瓷”主题的手机壳。
**流程**：
1.  Sarah 在画板上草绘了一个简单的缠枝莲纹样。
2.  呼叫 **创意艺匠 Agent**：“请把这个草图渲染成明代青花瓷风格，要注意釉下彩的晕染效果。”
3.  Agent 调用 **Stable Diffusion + ControlNet**，加载“青花瓷 LoRA”，生成了三张不同烧制效果的高清图。
4.  Sarah 选中一张，并询问 **知识馆长 Agent**：“这种纹样在明代通常寓意什么？”
5.  Agent 检索知识图谱，回答：“缠枝莲纹寓意生生不息、连绵不断，常用于……”

### 场景三：数字化展览互动 (Exhibiting)
**用户**：博物馆游客一家三口。
**流程**：
1.  在紫砂壶展区，孩子对着大屏幕体验“拍泥片”成型工艺。
2.  **视觉导师 Agent** 捕捉孩子的挥动手臂动作，屏幕上的虚拟泥片随之变化形状。
3.  完成后，系统生成一张“小小紫砂工匠”的电子证书，家长扫码保存并分享朋友圈。

## 4. 功能需求 (Functional Requirements)

### 4.1 核心功能模块

#### (1) 视觉导师 Agent (Vision-Mentor Agent) —— “手眼身法”
*   **功能描述**：利用计算机视觉技术，对用户的操作动作进行实时捕捉、分析与纠偏。
*   **关键特性**：
    *   **实时骨骼追踪**：支持手部 21 关键点 (MediaPipe Hands) 或全身姿态 (Pose) 识别。
    *   **动作比对**：将用户动作与“大师标准动作库”进行 DTW (动态时间规整) 或余弦相似度比对。
    *   **多维反馈**：提供视觉（骨骼线变色）、听觉（语音提示）双重反馈。
    *   **数据记录**：记录练习时长、动作准确率，生成学习曲线。

#### (2) 知识馆长 Agent (Knowledge-Curator Agent) —— “博古通今”
*   **功能描述**：基于 RAG (检索增强生成) 的智能问答助手，解答非遗相关的历史、工艺、材料问题。
*   **关键特性**：
    *   **知识图谱挂载**：集成非遗领域知识图谱 (Knowledge Graph)，确保实体关系准确（如：苏绣 -> 属于 -> 刺绣 -> 发源地 -> 苏州）。
    *   **多模态问答**：支持“拍图提问”（识别工具有什么用）和语音提问。
    *   **图文并茂**：回答中自动穿插相关的古籍图片、工具照片或演示视频片段。

#### (3) 创意艺匠 Agent (Creative-Artisan Agent) —— “妙笔生花”
*   **功能描述**：辅助用户进行非遗风格的创意设计与复原。
*   **关键特性**：
    *   **线稿生图**：基于 ControlNet (Scribble/Canny)，将用户简笔画转化为精细的非遗风格图像。
    *   **风格迁移**：提供多种非遗 LoRA 模型选择（如：苏绣、蜡染、剪纸、青花瓷）。
    *   **参数调节**：支持调节“做旧程度”、“材质质感”等专业参数。

### 4.2 系统管理功能
*   **主协调器 (Master Orchestrator)**：基于 MCP 协议，负责意图识别与任务分发。
*   **用户档案系统**：存储学习进度、作品集、能力雷达图。

## 5. 技术架构与创新点 (Technical Architecture)

### 5.1 技术栈
*   **前端**：React  + Three.js (3D展示) + TensorFlow.js (端侧推理)
*   **后端**：Python (FastAPI/Flask) + LangChain (Agent 编排)
*   **AI 模型**：
    *   视觉：MediaPipe, ST-GCN (时空图卷积网络)
    *   大模型：Qwen / GLM-4 (基座模型)
    *   生图：Stable Diffusion XL + LoRA
*   **数据存储**：Neo4j (知识图谱) + Milvus (向量数据库) + PostgreSQL (业务数据)

### 5.2 创新点 (Competition Highlights)
1.  **“多智能体协作”架构 (Multi-Agent)**：突破单一功能的局限，实现了视觉、语言、生成能力的有机融合，模拟真实的“师徒教学”场景。
2.  **端侧推理优化**：针对视觉模块进行轻量化处理 (TensorFlow.js / ONNX)，确保在普通移动设备上也能流畅运行，体现工程化能力。
3.  **非遗垂直领域知识库**：构建了专有的非遗知识图谱和动作数据集，解决了通用模型在特定领域“不懂行”的问题。

## 6. 非功能性需求 (Non-functional Requirements)
*   **响应速度**：视觉反馈延迟 < 100ms；对话响应 < 2s；生图响应 < 10s。
*   **易用性**：界面设计需符合东方美学，操作流程简单直观，适配触屏交互。
*   **兼容性**：支持主流浏览器 (Chrome, Edge) 及移动端 Web 访问。
*   **隐私安全**：用户摄像头数据仅在本地处理，不上传云端，保护用户隐私。
