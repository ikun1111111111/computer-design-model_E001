## 二、实现内容

### 2.1 数据模型设计

#### 1. 用户表 (users)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 用户 ID |
| name | VARCHAR(100) | NOT NULL | 用户姓名 |
| level | INT | DEFAULT 1 | 用户等级 (1-10) |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引设计**:
- `idx_users_name`: 基于 name 字段的索引，加速用户查询

#### 2. 学习记录表 (practice_logs)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 记录 ID |
| user_id | INT | FOREIGN KEY → users.id | 用户 ID |
| craft_id | INT | NOT NULL | 工艺 ID |
| craft_name | VARCHAR(100) | - | 工艺名称 |
| duration | INT | - | 练习时长 (秒) |
| score | DECIMAL(5,2) | - | 评分 (0-100) |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引设计**:
- `idx_practice_logs_user_id`: 加速按用户查询学习记录
- `idx_practice_logs_craft_id`: 加速按工艺查询记录
- 外键约束：`fk_practice_logs_user` (级联删除)

#### 3. 作品集表 (portfolios)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 作品 ID |
| user_id | INT | FOREIGN KEY → users.id | 用户 ID |
| image_url | VARCHAR(500) | NOT NULL | 作品图片 URL |
| craft_type | VARCHAR(100) | - | 工艺类型 |
| description | TEXT | - | 作品描述 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引设计**:
- `idx_portfolios_user_id`: 加速按用户查询作品集
- 外键约束：`fk_portfolios_user` (级联删除)

## 四、技术说明

### 4.1 设计亮点

1. **外键级联删除**: 删除用户时自动删除关联的学习记录和作品，保持数据一致性
2. **索引优化**: 为常用查询字段 (user_id, craft_id, name) 创建索引
3. **注释完善**: 所有字段都有中文 COMMENT，便于团队协作
4. **时间戳默认值**: 使用 `DEFAULT CURRENT_TIMESTAMP` 自动记录创建时间
5. **UTF8MB4 编码**: 支持完整的 Unicode 字符集，包括 emoji

### 4.2 扩展性考虑

- `level` 字段预留用户等级功能，支持后续成长体系
- `craft_id` 和 `craft_name` 分离设计，支持工艺库动态扩展
- `portfolios.description` 使用 TEXT 类型，支持长文本描述
- 所有表预留 `created_at` 字段，支持时间维度分析
