针对你选择的“数字传承人”——非遗工艺流程 Agent 交互教学与复原助手，这不仅契合了 2025 年“中国古代物理/中华优秀传统文化”的大赛主题，更在技术上精准踩中了“多智能体协作（Multi-Agent）”和“多模态交互”的热点。

以下是该项目的详细设计方案及其对应的需求文档（PRD）草案。

### 1. 技术架构设计方案

该系统采用“1个协调中心 + 3个专业代理”的层次化多智能体架构（Hierarchical Multi-Agent Pattern）。

#### 核心代理角色定义：

1. **视觉导师 Agent (Vision-Mentor Agent)**：
* **核心功能**：利用计算机视觉技术实时采集并分析用户的操作手势（如苏绣的“拨针”、紫砂的“拍泥片”）。
* **技术栈**：基于 **MediaPipe** 或 **ST-GCN (时空图卷积网络)** 进行骨骼关键点追踪，通过与专家库中的“大师手势数据集”比对，计算欧氏距离或余弦相似度，实时给出“力度过大”或“角度偏低”的反馈。


2. **知识馆长 Agent (Knowledge-Curator Agent)**：
* **核心功能**：解答关于非遗工艺的历史、材料属性及美学意蕴的深度提问。
* **技术栈**：采用 **RAG (检索增强生成)** 技术，挂载非遗知识图谱（如 CICHMKG）。通过 **Neo4j** 存储非遗传承脉络，确保回答的准确性和权威性，避免大模型的“幻觉”。


3. **创意艺匠 Agent (Creative-Artisan Agent)**：
* **核心功能**：将用户的草图（Sketch）转化为符合特定非遗风格（如苏绣质感、青花瓷纹样）的数字化作品。
* **技术栈**：基于 **Stable Diffusion XL + ControlNet (Scribble/Canny)** 模型。通过加载特定非遗技法微调的 **LoRA** 模型（如“苏绣质感 LoRA”或“蓝夹缬风格 LoRA”），实现从“随手涂鸦”到“大师意境”的跨越。



#### 协作逻辑：

系统引入 **Master Orchestrator (主协调器)**，使用 **MCP (Model Context Protocol)** 协议进行标准化通信。当用户说“我想学这个针法并画一个样稿”时，主协调器会自动拆解任务：由知识 Agent 讲解原理，视觉 Agent 进入监控模式，生成 Agent 准备画布。

### 2. 产品需求文档 (PRD)

#### 2.1 项目概述

* **项目名称**：数字传承人 (The Digital Inheritor)
* **目标赛道**：软件应用与开发 —— Web 应用与开发 / 移动应用开发
* **愿景**：通过 AI Agent 填补非遗技艺中“言传身教”的数字化空白，实现低门槛、高互动的非遗传承教学。

#### 2.2 用户画像与场景

| 用户角色 | 使用场景 | 核心需求 |
| --- | --- | --- |
| **高校学生/初学者** | 艺术选修课或非遗工作坊 | 希望通过手机或平板即时纠正练习动作，避免误操作造成材料浪费。 |
| **文化创意设计者** | 品牌联名或国潮产品设计 | 需要将简单的设计草图快速转化为具有非遗肌理的渲染图作为打样参考。 |
| **非遗博物馆/展馆** | 游客交互式体验区 | 提供“云拜师”体验，增加数字化展览的趣味性与交互性。 |

#### 2.3 核心功能需求 (Functional Requirements)

1. **实时手势纠偏系统 (Vision Control)**：
* 支持 RGB 摄像头接入，实时提取手部 21 个关键点。
* 动态对比阈值：动作相似度低于 75% 时触发语音或视觉提示。


2. **多模态非遗问答 (Knowledge Q&A)**：
* 支持语音输入与文本输入，提供语义准确的非遗科普。
* 具备“图文并茂”能力，在讲解时自动调取相关的历史古籍图片。


3. **AIGC 风格复原 (Creative Synthesis)**：
* 输入：手绘线稿 + 提示词。
* 输出：高清非遗风格图。支持针法/烧制纹理/上色工艺等参数调节。


4. **学习档案与能力评分 (Performance Metrics)**：
* 记录用户的练习时长、动作准确率变化趋势。
* 生成“数字传承人成长证书”，增强成就感。



#### 2.4 非功能性需求 (Non-functional Requirements)

* **实时性 (Latency)**：视觉识别 Agent 的端到端延迟需控制在 100ms 以内，以保证纠偏的及时性。
* **准确性**：知识问答在特定非遗领域的 RAG 召回准确率应高于 85%。
* **规范性**：符合国家非遗保护的相关法律法规，地图及文化符号引用须准确合规。

#### 2.5 成功度量标准 (Success Metrics)

* **技术维度**：多智能体协作的闭环成功率达到 100%。
* **应用维度**：相比传统视频教学，用户对复杂动作的掌握时间缩短 35% 以上。

### 3. “技术流”参赛获胜关键建议

1. **工程化文档**：在决赛答辩中，务必展示你的 **Agent 状态机切换图** 和 **MCP 通信协议日志**，这能体现你作为技术选手的严谨性。
2. **数据集创新**：如果能自己录制一小段本地特有的非遗大师动作（即使只有几分钟）并进行标注，会比直接调用通用开源模型更具行业说服力。
3. **模型轻量化**：对于视觉 Agent，强调你在移动端或边缘侧（如使用 **NIM 推理服务**）的优化，确保普通手机也能流畅运行，这在实务类作品评审中是重大的加分项。
